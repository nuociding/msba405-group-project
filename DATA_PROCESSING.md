# Data Processing Pipeline

This document describes the data preparation pipeline used in the project.

All tract-level datasets used in the analysis are generated by running the notebook `data_processing.ipynb`.

---

# Raw Data Sources

The project integrates several public datasets for Los Angeles census tract–level analysis.  
These datasets are combined to construct tract-level features for the Whole Foods location recommendation model.

Data sources include:

- **Los Angeles Crime Data** — City of Los Angeles Open Data Portal  
- **ACS Demographic Data** — American Community Survey (population, income, education)  
- **TIGER/Line Census Tracts** — U.S. Census Bureau tract boundary shapefiles  
- **OpenStreetMap (OSM) Points** — retail and service amenities  
- **Whole Foods Store Locations** — store coordinates in Los Angeles  

---

# Processing Steps

The notebook `data_processing.ipynb` performs the following pipeline steps:

1. Load all raw datasets
2. Prepare Los Angeles census tract boundaries
3. Clean and preprocess crime records
4. Spatially assign crime points, OSM amenities, and Whole Foods locations to census tracts
5. Construct tract-level demographic features from ACS data
6. Compute density-based features using tract land area
7. Generate tract-level feature tables for downstream modeling

---

# Output Datasets

Running `data_processing.ipynb` produces the following tract-level datasets:

- `tract_crime.csv`
- `tract_acs.csv`
- `tract_osm.csv`
- `tract_wf.csv`

These datasets are used for downstream feature engineering and model scoring.

Due to GitHub storage limitations, processed datasets are not included in this repository.
